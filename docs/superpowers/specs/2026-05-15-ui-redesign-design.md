# UI 重构设计文档

## 目标

将 IELTS 练习系统的 UI 重构为 Material 3 动态风格，实现炫酷高级的视觉效果，包括深色主题、毛玻璃卡片、丰富微交互和数据可视化。

## 设计系统

### 配色方案（深色主题）

- **背景**：`#0f172a` → `#1e293b` 渐变
- **卡片背景**：`rgba(255,255,255,0.05)` 毛玻璃效果
- **主色调**：`#6366f1`（靛蓝）+ `#8b5cf6`（紫色）渐变
- **成功色**：`#22c55e`
- **错误色**：`#ef4444`
- **警告色**：`#f59e0b`
- **文字**：`#e2e8f0`（主）、`#94a3b8`（次）、`#64748b`（辅助）

### 字体系统

- **标题**：`Inter`（现代无衬线）
- **正文**：`Inter` + `Noto Sans SC`（中文支持）
- **代码**：`JetBrains Mono`
- **字体权重**：400（正常）、500（中等）、600（半粗）、700（粗体）

### 间距系统

基于 8px 网格：
- `--space-1`: 4px
- `--space-2`: 8px
- `--space-3`: 12px
- `--space-4`: 16px
- `--space-6`: 24px
- `--space-8`: 32px
- `--space-12`: 48px
- `--space-16`: 64px

### 圆角系统

- 小元素（按钮、标签）：8px
- 卡片、面板：16px
- 按钮：12px
- 头像/图标：50%
- 完全圆形：9999px

### 阴影系统

- **卡片**：`0 8px 32px rgba(0,0,0,0.4)`
- **悬浮**：`0 12px 40px rgba(99,102,241,0.3)`
- **弹窗**：`0 24px 64px rgba(0,0,0,0.5)`

## 页面布局

### 整体结构

- **最大宽度**：1200px，居中显示
- **背景**：深色渐变（`#0f172a` → `#1e293b`）+ CSS 噪点纹理（`background-image: url("data:image/svg+xml,...")` 生成的微妙噪点）
- **内容区域**：左右 padding 24px，响应式设计

### 顶部导航

- **位置**：固定在顶部
- **背景**：毛玻璃效果（`backdrop-filter: blur(20px)`）
- **Tab 切换**：滑动指示器动画
- **当前 Tab**：发光效果（`box-shadow: 0 0 20px rgba(99,102,241,0.5)`）

### 题库页面

- **顶部**：搜索框 + 筛选按钮（全部/阅读/听力）
- **中间**：卡片网格（桌面端 3 列）
- **卡片**：毛玻璃效果，悬浮时上浮 + 发光边框

### 错题本页面

- **顶部**：筛选 + 排序 + 批量操作
- **列表**：详细卡片设计
- **展开详情**：题目原文预览 + 重做按钮

## 组件设计

### 考试卡片

- **背景**：毛玻璃效果（`rgba(255,255,255,0.05)`）
- **边框**：1px solid rgba(255,255,255,0.1)
- **圆角**：16px
- **悬浮效果**：上浮 8px + 发光边框 + 阴影增强
- **内容**：标题、分类、频率、练习次数

### 错题卡片

- **背景**：渐变背景（靛蓝到紫色，10% 透明度）
- **边框**：1px solid rgba(99,102,241,0.3)
- **圆角**：16px
- **内容**：题目、答案对比、重做状态、操作按钮

### 按钮

- **主要按钮**：渐变背景（靛蓝到紫色）
- **次要按钮**：透明背景 + 边框
- **悬浮**：背景变亮
- **点击**：涟漪效果（Material 风格）
- **禁用**：半透明 + 禁止光标

### 搜索框

- **背景**：`rgba(255,255,255,0.05)`
- **边框**：1px solid rgba(255,255,255,0.1)
- **聚焦**：发光边框（`box-shadow: 0 0 0 2px rgba(99,102,241,0.5)`）
- **占位符**：`#64748b`

## 动画效果

### 页面切换

- **Tab 切换**：内容淡入淡出（300ms）
- **页面加载**：元素依次入场（stagger 动画，每项延迟 50ms）

### 卡片交互

- **悬浮**：上浮 8px + 发光边框 + 阴影增强（200ms）
- **点击**：涟漪效果（300ms）
- **展开/折叠**：平滑高度过渡（300ms）

### 按钮交互

- **悬浮**：背景变亮（150ms）
- **点击**：涟漪效果（300ms）
- **禁用**：半透明 + 禁止光标

### 加载状态

- **骨架屏**：卡片加载时显示骨架动画（1.5s 循环）
- **按钮加载**：旋转图标（1s 循环）
- **页面加载**：全屏加载动画

### Toast 提示

- **位置**：底部居中
- **动画**：从底部滑入（300ms）
- **自动消失**：3 秒后淡出
- **类型**：成功（绿色）、错误（红色）、警告（黄色）

### 视差滚动

- **背景**：使用 CSS `transform: translateZ()` 和 `perspective` 实现视差效果，滚动时背景移动速度为内容的 0.5 倍
- **内容**：正常滚动

## 数据可视化

### 技术选型

- **库**：Chart.js（轻量级，~60KB）
- **主题**：深色主题，自定义颜色
- **响应式**：自适应容器宽度

### 环形正确率图

- **位置**：题库页面顶部或侧边
- **数据**：总体正确率
- **样式**：中心显示百分比，颜色渐变（绿到红）
- **动画**：加载时从 0 增长到实际值

### 每日练习柱状图

- **位置**：题库页面统计区域
- **数据**：最近 7 天练习量
- **样式**：柱状图有渐变色，悬浮显示具体数值
- **动画**：加载时从底部增长

### 学习趋势折线图

- **位置**：错题本页面统计区域
- **数据**：正确率变化趋势
- **样式**：平滑曲线，关键点标记
- **动画**：加载时从左到右绘制

## 技术实现

### 技术栈

- HTML5 + CSS3 + ES6+ JavaScript
- Chart.js（图表库）
- CSS 变量实现动态主题
- CSS 动画（优先使用 transform 和 opacity）

### 文件结构

```
css/
├── variables.css      # 设计 tokens
├── base.css          # 重置和基础样式
├── components/       # 组件样式
│   ├── card.css
│   ├── button.css
│   ├── navigation.css
│   ├── search.css
│   ├── filter.css
│   ├── toast.css

├── animations/       # 动画定义

│   ├── keyframes.css
│   └── ripple.css
└── main.css          # 主样式入口

js/
├── charts/           # 图表相关代码
│   ├── accuracy.js
│   ├── practice.js
│   └── trend.js
├── animations/       # 动画相关代码
│   ├── ripple.js
│   └── toast.js
└── utils/            # 工具函数
    └── dom.js
```

### 性能优化

- 使用 CSS `will-change` 提示浏览器优化
- 动画优先使用 `transform` 和 `opacity`
- 图表懒加载（Intersection Observer）
- 图片懒加载

### 兼容性

- 现代浏览器（Chrome、Firefox、Safari、Edge）
- 不支持 IE

## 实现计划

### 实施顺序

1. **设计系统基础**（2-3 小时）
   - CSS 变量定义
   - 字体导入
   - 间距系统

2. **页面布局**（3-4 小时）
   - HTML 结构重构
   - 响应式设计
   - 导航组件

3. **组件样式**（4-5 小时）
   - 卡片组件
   - 按钮组件
   - 搜索框
   - 筛选器

4. **动画效果**（3-4 小时）
   - 悬浮效果
   - 涟漪动画
   - 页面切换
   - 加载状态

5. **数据可视化**（2-3 小时）
   - Chart.js 集成
   - 环形正确率图
   - 每日练习柱状图
   - 学习趋势折线图

6. **交互优化**（2-3 小时）
   - Toast 提示
   - 骨架屏
   - 视差滚动

### 总计工作量

16-22 小时

### 测试策略

- 手动测试各页面和交互
- 验证响应式设计
- 检查动画性能
- 跨浏览器测试
